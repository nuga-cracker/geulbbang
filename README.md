# 🍞 글빵 (GeulBbang)

> 맛있는 글을 만들어주는 제빵소 — 글쓰기 첨삭 + 성장형 게임

투박한 초고(반죽)를 첨삭 피드백에 따라 여러 번 고쳐 쓰면서, 점점 더 좋은 빵(유려한 글)으로 구워내는 성장 게임입니다. 🥖

**💸 유지비 0원** — 서버·API·데이터베이스가 전혀 없고, 브라우저 안에서만 동작합니다.

---

## 🍞 컨셉 (글쓰기 = 제빵)

| 제빵 과정 | 글쓰기 과정 |
|-----------|-------------|
| 밀가루/반죽 | 처음 쓴 초고 (거칠고 투박한 날글) |
| 반죽 치대기 | 첨삭(선생님 피드백) 받기 |
| 발효 | 피드백을 반영해 다시 쓰기 |
| 굽기 | 여러 번 고쳐서 문장이 익어감 |
| 완성된 빵 | 잘 다듬어진 유려한 글 |

## ✨ 주요 기능

### 1. 글쓰기 & 첨삭
- 📝 텍스트 영역에 글을 입력하고 **"빵 굽기(첨삭받기)"** 버튼 클릭
- 👨‍🍳 **선생님(제빵사) 말투**로 다정한 피드백 — 칭찬 + 구체적 개선점 + 격려
- 📊 글자 수 · 단어 수 · 문장 수 통계 및 **맛있음 점수(0~100점)** 산출

### 2. 규칙 기반 첨삭 엔진 (외부 API 없음)
- 🔴 **긴 문장** — 60자 이상 문장 분리 제안
- 🟡 **중복 단어** — 같은 단어 3회 이상 반복 감지
- 🟡 **군더더기 표현** — "그리고", "그래서", "정말", "너무", "~것 같다" 등 남용 감지
- 🔵 **피동 표현 / 이중 부정** 패턴 힌트
- 🔵 **존댓말/반말 혼용** 일관성 체크
- 각 지적마다 *왜 고치면 좋은지* 아이 눈높이로 설명

### 3. 성장형 게임
- ⭐ **경험치(EXP) & 레벨** — 쓰고 고칠수록 EXP 획득, 재작성 시 보너스 대폭 증가
- 🍞 **빵 등급 진화** — 반죽 🫙 → 식빵 🍞 → 모닝빵 🥐 → 크루아상 🥐 → 바게트 🥖 → 명품 바게트 🥖✨
- 🏅 **배지 / 도전과제** — "첫 빵 완성", "3일 연속", "한 글을 5번 고쳐쓰기" 등 20종 (문체 관련 배지 포함)
- 🗂️ **진열장(포트폴리오)** — 완성한 글을 등급·점수·문체·날짜와 함께 보관
- 🌅 **오늘의 글감** — 매일 바뀌는 글쓰기 주제 30종 내장

### 4. 🍽️ 작가 문체 레시피 시스템 (2단계)

글을 쓰기 전 **"오늘은 어떤 작가처럼 구울까요?"** 섹션에서 6종의 문체 레시피 중 하나를 선택하면, 기존 규칙 첨삭 결과에 **선택한 문체 방향 코멘트**가 선생님 말투로 함께 표시됩니다.

| 레시피 | 분위기 | 기반 |
|--------|--------|------|
| 🌫️ 부끄러운 고백빵 | 솔직한 1인칭 고백, 나직한 독백 | 다자이 오사무 참고 (저작권 만료) |
| 🌸 노래하는 진달래빵 | 서정적, 민요조 리듬, 그리움 | 김소월 참고 (저작권 만료) |
| 😄 구수한 시골빵 | 해학, 토속어, 익살, 정겨운 시골 | 김유정 참고 (저작권 만료) |
| 😏 재치있는 티타임빵 | 위트, 우아함, 은근한 풍자 | 제인 오스틴 참고 (저작권 만료) |
| 🌸 매일 다정 일기빵 | 따뜻함, 일상의 소중함, 다정한 구어체 | 순수 창작 (다정한 일상 일기 스타일) |
| 🕯️ 차분 사색빵 | 고요함, 깊이 있는 사색, 여백 있는 담백함 | 순수 창작 프리셋 |

#### 🛡️ 저작권 안전 원칙

- **저작권 만료(퍼블릭 도메인) 작가**만 실제 문체 특징과 아주 짧은 예시 문구를 활용합니다. 기준: 저작자 사후 70년 경과 작가.
- **현대(생존) 작가의 원문(문장)은 일절 저장·인용·복제하지 않습니다.** 현대 작가는 이름을 직접 표방하지 않으며, "느낌·분위기"만 순수 창작 프리셋으로 추상화했습니다.
- 각 레시피에는 원문 전체가 아니라 **문체의 특징(문장 길이, 어미, 어조, 정서, 어휘 경향 등)**을 규칙·설명 형태로만 담았습니다.

### 5. 데이터 저장
- 💾 모든 진행 상황을 브라우저 **localStorage**에 자동 저장
- 새로고침·재방문해도 레벨·배지·진열장 유지
- 데이터 초기화 버튼 제공

### 6. 🎨 3단계 — 나만의 문체(시그니처 빵) + 온보딩/가이드

- 🏆 **나만의 시그니처 빵 레시피**
  - (A) 설문으로 빠르게 초안 만들기
  - (B) 내가 쓴 글 자동 분석(평균 문장 길이, 자주 쓰는 단어, 말투 경향, 물음표/느낌표 습관)
  - (C) 슬라이더(문장 길이/표현 밀도/활발함/유머)로 미세 조정
- 🍽️ 만든 시그니처 빵은 기존 작가 레시피와 똑같이 선택 가능하며, 첨삭 결과에 문체 방향 코멘트가 함께 표시됩니다.
- 🧭 **글쓰기 플로우 가이드**: "쓰기 → 빵 굽기(첨삭) → 고쳐쓰기 → 진열장 저장" 흐름을 화면에서 바로 안내합니다.
- 👋 **첫 방문 온보딩(5단계)**: 다음/이전/건너뛰기 지원, 한 번 완료하면 자동으로 숨김, 도움말(❓) 버튼으로 언제든 다시 보기 가능.
- 💾 온보딩 완료 여부, 시그니처 빵 설정, 문체 분석 기록도 localStorage에 함께 저장됩니다.

## 🚀 로컬에서 실행하는 법

**방법 1 — 그냥 파일 열기 (가장 간단)**

1. 이 저장소를 다운로드하거나 클론합니다.
2. `index.html` 파일을 더블클릭하거나 브라우저로 엽니다.
3. 끝! 서버·빌드·설치 모두 불필요합니다.

> ⚠️ 단, `<script type="module">` 때문에 일부 브라우저에서 `file://` 프로토콜로 열면 CORS 오류가 날 수 있어요.
> 그럴 땐 방법 2를 사용하세요.

**방법 2 — 로컬 서버 사용 (VS Code 추천)**

```bash
# VS Code의 Live Server 확장 설치 후 index.html에서 우클릭 → "Open with Live Server"
# 또는 Python으로 간단한 서버 실행:
python -m http.server 8080
# 브라우저에서 http://localhost:8080 접속
```

## 🌐 GitHub Pages로 무료 배포하는 방법

1. GitHub에서 이 저장소를 본인 계정으로 Fork하거나, 새 저장소에 코드를 push합니다.
2. 저장소 페이지 상단 **Settings** 탭 클릭
3. 왼쪽 메뉴에서 **Pages** 클릭
4. **Source** 섹션에서 **"Deploy from a branch"** 선택
5. Branch: **main**, 폴더: **/ (root)** 선택 후 **Save**
6. 약 1~2분 후 `https://<유저명>.github.io/<저장소명>/` 주소로 접속 가능!

> ✅ 완전 무료 · 광고 없음 · HTTPS 자동 적용

## 🤖 향후 AI 연동 확장 가능성

`js/checker.js`의 `analyzeText()` 함수 상단에 **AI 연동 스텁(주석)**이 준비되어 있습니다.

```js
// js/checker.js 안에 있는 AI 연동 포인트:
// if (window.AI_MODE && window.OPENAI_API_KEY) {
//   return analyzeWithAI(text, revisionCount);
// }
```

나중에 OpenAI 등 AI API를 붙이고 싶으면:
1. `window.AI_MODE = true`와 `window.OPENAI_API_KEY = '키'` 설정
2. `analyzeWithAI()` 함수 구현 후 스텁 활성화

**현재는 이 코드가 실행되지 않아 외부 API를 전혀 호출하지 않습니다.** 🔒
