# 글빵(GeulBbang) 사용설명서

> 글빵은 초고를 “반죽”처럼 넣고, 첨삭으로 문장을 치대고, 다시 고쳐 쓰며 나만의 글을 구워내는 글쓰기 연습 앱입니다.

---

## 1. 접속하기

GitHub Pages 배포 주소로 접속합니다.

```text
https://nuga-cracker.github.io/geulbbang/
```

화면이 예전 버전처럼 보이면 브라우저 캐시 때문일 수 있습니다.

- Windows: `Ctrl + F5`
- Mac: `Cmd + Shift + R`
- 또는 주소 뒤에 `?v=latest`를 붙여 접속

```text
https://nuga-cracker.github.io/geulbbang/?v=latest
```

---

## 2. 처음 방문했을 때

처음 들어오면 온보딩 안내가 표시됩니다.

온보딩에서는 다음 내용을 알려줍니다.

1. 글쓰기 기본 흐름
2. 빵 굽기와 첨삭 확인
3. 문체 레시피 사용법
4. 나만의 시그니처 문체 만들기
5. 진열장과 성장 요소

온보딩은 건너뛸 수 있고, 나중에 상단의 도움말 버튼으로 다시 볼 수 있습니다.

---

## 3. 기본 사용 흐름

글빵의 핵심 흐름은 아래와 같습니다.

```text
글쓰기
→ 빵 굽기
→ 첨삭 확인
→ 비문 다듬기 확인
→ 제안 적용하기
→ 다시 빵 굽기
→ 마음에 들면 진열장 저장
```

---

## 4. 글 쓰기

1. `글쓰기` 화면으로 이동합니다.
2. 글쓰기 칸에 초고를 입력합니다.
3. 짧은 일기, 감상문, 설명문, 자기소개, 메모 등 어떤 글이든 넣을 수 있습니다.
4. 글을 입력한 뒤 `빵 굽기` 버튼을 누릅니다.

예시 문장:

```text
나는은 오늘 학교에에 갔다. 그리고 그리고 친구를 만났다.
```

---

## 5. 빵 굽기

`빵 굽기`를 누르면 글빵이 문장을 분석합니다.

분석 후 확인할 수 있는 내용은 다음과 같습니다.

- 글 점수
- 격려 문구
- 문장 길이 관련 피드백
- 반복어 피드백
- 군더더기 표현 피드백
- 존댓말/반말 섞임 안내
- 비문/어색한 문장 다듬기
- 선택한 문체 레시피에 따른 조언

---

## 6. 기본 첨삭 확인하기

기본 첨삭에서는 문장을 더 읽기 좋게 만드는 피드백을 제공합니다.

예를 들어 다음과 같은 내용을 확인할 수 있습니다.

- 문장이 너무 길어요.
- 같은 단어가 여러 번 반복됐어요.
- `정말`, `너무`, `그리고` 같은 표현이 많아요.
- 존댓말과 반말이 섞여 있을 수 있어요.

피드백은 틀렸다고 혼내는 방식이 아니라, 더 자연스럽게 고쳐볼 수 있도록 안내하는 방식입니다.

---

## 7. 비문 다듬기 사용하기

`🧁 비문 다듬기` 섹션에서는 어색한 한국어 문장이나 반복 표현을 찾아줍니다.

### 감지 예시

| 원문 | 안내 |
|---|---|
| `나는은` | 조사가 겹쳐 어색할 수 있어요. |
| `학교에에` | 같은 조사가 반복됐어요. |
| `그리고 그리고` | 연결어가 반복됐어요. |
| `했다 했다` | 종결 표현이 반복됐어요. |
| `에 있어서` | 조금 딱딱한 번역투일 수 있어요. |
| `하는 것이 좋을 것 같다` | 문장이 장황할 수 있어요. |

---

## 8. `이 제안 적용하기` 버튼 사용하기

비문 다듬기 중 자동으로 고쳐도 안전한 항목에는 `이 제안 적용하기` 버튼이 표시됩니다.

### 사용 방법

1. 글을 입력합니다.
2. `빵 굽기`를 누릅니다.
3. `🧁 비문 다듬기` 섹션을 확인합니다.
4. `이 제안 적용하기` 버튼을 누릅니다.
5. 글쓰기 칸의 문장이 자동으로 수정됩니다.
6. 다시 `빵 굽기`를 눌러 개선된 결과를 확인합니다.

### 자동 적용 예시

| 원문 | 적용 후 |
|---|---|
| `나는은` | `나는` |
| `학교에에` | `학교에` |
| `친구가가` | `친구가` |
| `책을를` | `책을` |
| `그리고 그리고` | `그리고` |
| `했다 했다` | `했다` |
| `입니다 입니다` | `입니다` |

### 자동 적용하지 않는 경우

문맥에 따라 다르게 고쳐야 하는 표현은 버튼이 표시되지 않을 수 있습니다.

예:

- `에 있어서`
- `에 대하여`
- `을 통해서`
- 장황한 표현
- 문맥을 봐야 하는 표현

이런 항목은 안내를 참고해서 직접 고치는 것이 좋습니다.

---

## 9. 문체 레시피 선택하기

글빵에는 여러 가지 문체 레시피가 있습니다.

문체 레시피를 선택하면 기본 첨삭에 더해, 선택한 분위기에 맞는 글쓰기 조언을 받을 수 있습니다.

### 레시피 예시

- 🌫️ 부끄러운 고백빵
- 🌸 노래하는 진달래빵
- 😄 구수한 시골빵
- 😏 재치있는 티타임빵
- 🌸 매일 다정 일기빵
- 🕯️ 차분 사색빵

### 사용 방법

1. 글쓰기 화면에서 원하는 레시피 카드를 선택합니다.
2. 글을 입력합니다.
3. `빵 굽기`를 누릅니다.
4. 기본 첨삭과 함께 문체 조언을 확인합니다.

---

## 10. 나만의 시그니처 문체 만들기

`나만의 시그니처 빵` 기능을 사용하면 내 취향에 맞는 문체를 만들 수 있습니다.

### 만드는 방법

1. 문체 만들기 영역으로 이동합니다.
2. 설문을 작성합니다.
   - 원하는 분위기
   - 문장 길이
   - 말투
   - 자주 쓰고 싶은 단어
3. 내 글 분석을 실행해 문체 지문을 반영합니다.
4. 슬라이더로 문체를 미세 조정합니다.
5. 완성된 문체를 레시피처럼 선택해 사용합니다.

### 활용 예시

- 내 글을 더 다정하게 만들기
- 문장을 더 짧고 선명하게 만들기
- 감정 표현을 더 풍부하게 만들기
- 나만의 말투를 유지하면서 첨삭 받기

---

## 11. 진열장에 저장하기

마음에 드는 글은 진열장에 저장할 수 있습니다.

### 저장되는 정보

- 글 내용
- 점수
- 빵 등급
- 사용한 문체 레시피
- 저장 시점의 결과

### 사용 방법

1. 글을 씁니다.
2. `빵 굽기`를 누릅니다.
3. 결과가 마음에 들면 저장 버튼을 누릅니다.
4. 진열장 탭에서 저장된 글을 확인합니다.

진열장은 내가 구운 글을 모아두는 공간입니다.

---

## 12. 배지와 레벨

글빵은 글쓰기 활동을 게임처럼 이어갈 수 있도록 배지와 레벨을 제공합니다.

### 얻을 수 있는 보상

- EXP
- 레벨 상승
- 빵 등급 변화
- 활동 배지

### 배지를 얻는 예시

- 첫 글을 구웠을 때
- 글을 여러 번 수정했을 때
- 문체 레시피를 처음 사용했을 때
- 여러 레시피를 사용했을 때
- 나만의 문체를 완성했을 때
- 문체 분석을 실행했을 때

---

## 13. 캐릭터 안내

화면 곳곳에 빵 캐릭터와 반죽 캐릭터가 등장합니다.

캐릭터는 기능을 방해하지 않는 장식 요소이며, 글빵의 따뜻한 제빵소 분위기를 더해줍니다.

---

## 14. 저장 데이터와 초기화

글빵은 서버에 데이터를 저장하지 않습니다.

대신 사용자의 브라우저 `localStorage`에 다음 정보를 저장합니다.

- 작성 기록
- EXP/레벨
- 배지
- 진열장 글
- 선택한 레시피
- 나만의 문체 설정
- 온보딩 완료 여부

브라우저 캐시나 사이트 데이터를 삭제하면 저장된 정보가 사라질 수 있습니다.

## 16. 테스트용 문장

기능을 빠르게 확인하려면 아래 문장을 입력해보세요.

```text
나는은 오늘 학교에에 갔다. 그리고 그리고 친구를 만났다. 이 문제에 있어서 우리는 많은 생각을 하는 것이 좋을 것 같다.
```

기대 결과:

- `나는은`, `학교에에`, `그리고 그리고`에 대한 비문 다듬기 제안 표시
- 일부 항목에 `이 제안 적용하기` 버튼 표시
- 버튼 클릭 시 글쓰기 칸의 문장이 자동 수정
- `에 있어서`, `하는 것이 좋을 것 같다`는 문맥에 따라 직접 고치도록 안내

## 17. 자주 묻는 질문

### Q. 글빵은 AI인가요?

현재 글빵은 외부 AI 없이 규칙 기반 JavaScript로 동작합니다. 정해진 문장 패턴과 통계를 바탕으로 피드백을 제공합니다.

### Q. 내 글이 서버에 저장되나요?

아니요. 서버나 DB를 사용하지 않습니다. 데이터는 사용자의 브라우저에 저장됩니다.

### Q. 피드백이 항상 맞나요?

아니요. 규칙 기반 피드백이므로 문맥에 따라 맞지 않을 수 있습니다. 글빵의 제안은 정답이 아니라 “고쳐볼 방향”으로 참고하는 것이 좋습니다.

### Q. 왜 어떤 제안에는 적용 버튼이 없나요?

자동으로 고치면 문맥이 이상해질 수 있는 표현은 버튼을 표시하지 않습니다. 안전하게 바꿀 수 있는 중복 조사, 반복 연결어 같은 항목에만 적용 버튼을 제공합니다.

### Q. 휴대폰에서도 사용할 수 있나요?

네. GitHub Pages 주소로 접속하면 모바일 브라우저에서도 사용할 수 있습니다.
